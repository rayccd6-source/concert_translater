演唱會即時翻譯 - 使用說明

把麥克風收到的語音（主持人中文，或翻譯員的泰/越/印/菲），即時翻譯並顯示成泰文、越南文、印尼文、菲律賓文四格字幕，同步投影在大螢幕上。

運作流程

  1. 後端用 sounddevice 持續從麥克風讀音訊
  2. 用 WebRTC VAD 判斷「現在有沒有人在講話」，把每一句話切出來
  3. 用 Faster-Whisper 本地模型把語音轉成文字（中文/越南文自動偵測）
  4. 把文字（不是音檔）丟給 Gemini，一次翻譯成 4 種語言
  5. 透過 WebSocket 把結果推到前端網頁，字幕即時更新
為什麼比原本 Hukou 專案快？因為原本是把整段 WAV 音檔丟給 Gemini 同時做識別+翻譯，音檔 token 非常貴又慢。改成「本地識別 + 雲端翻譯」後，token 用量降到約原本的 1/100，延遲也從幾秒降到 1 秒左右。

安裝步驟

1. 安裝 Python 套件

cd projects/concert-translator
pip install -r requirements.txt

第一次執行時，Faster-Whisper 會自動下載模型（small 約 250 MB），下載完會快取，之後不會再下載。

2. 設定 Gemini API Key

到 Google AI Studio 申請一把 API key（免費），然後：

cd backend
cp .env.example .env
# 用編輯器打開 .env，把 GEMINI_API_KEY=your_api_key_here 換成你的 key

3. 啟動後端

cd backend
python main.py

看到「服務啟動」訊息後，瀏覽器打開 http://localhost:8000 就會看到字幕頁面。

前端字幕頁面

瀏覽器分成兩區：

  • 頂部窄條：原文（中文或越南文）
  • 下方 2×2 網格：泰、越、印、菲 四種翻譯

建議按 F11 進入全螢幕模式投影到現場大螢幕。右上角小圓點綠色＝WebSocket 已連線。

調整參數

所有參數集中在 backend/.env 與 backend/config.py：

參數說明建議值
WHISPER_MODEL語音識別模型large-v3-turbo（推薦）／medium／small
WHISPER_CPU_THREADS使用幾個 CPU 核心0 = 全部（建議）
WHISPER_LANGUAGE強制指定語言：zh / vi / auto中文場次 zh、越南文場次 vi
WHISPER_DEVICEcpu 或 cuda有 NVIDIA 顯卡用 cuda 速度倍增
VAD_AGGRESSIVENESS切句敏感度 0~32（演唱會吵雜環境建議 3）
MIN_SPEECH_DURATION_MS短於這個的語音丟棄1000 ms（過短易誤判語言）
SILENCE_TIMEOUT_MS靜音多久判定一句話結束700 ms
MAX_SPEECH_DURATION_MS單句話最長秒數8000 ms
AUDIO_QUEUE_MAX音訊待處理隊列上限5（滿了自動丟最舊）
GEMINI_MODEL翻譯模型gemini-2.5-flash-lite（免費版 RPM 較寬）
TRANSLATOR_WORKERS翻譯並行數2（免費版）／4~6（付費版）

Whisper 模型怎麼選

關鍵認知：CPU 上沒有「又快又準」的解。所有 large 系列（包含 turbo）的 encoder 都很大，CPU 跑 encoder 是瓶頸，turbo 砍 decoder 的速度提升只有 GPU 上才看得出來。

CPU 場景（沒有 NVIDIA 顯卡）

模型中文精度CPU RTF (12 核)適用
medium好~1.5x（略慢於 realtime）CPU 最佳選擇
small差（中文短句易胡言亂語）~0.5x機器很弱才用
large-v3-turbo近 large~5x（太慢）不要在 CPU 用
large-v3最好~6x（太慢）不要在 CPU 用

GPU 場景（NVIDIA 顯卡 + WHISPER_DEVICE=cuda）

模型GPU RTF適用
large-v3-turbo0.1~0.3x（極快）GPU 最佳選擇
large-v30.3~0.5x追求最高精度

演唱會場景強烈建議：上 NVIDIA 顯卡

即使是入門卡（GTX 1660 / RTX 3050），跑 turbo 都能輕鬆即時。CPU 場景下中文翻譯精度與延遲注定有一個要妥協。

切換語言

預設 zh，越南文場次在 backend/.env 加 WHISPER_LANGUAGE=vi。

字幕顯示行為

四等分網格內每個語言獨立堆疊：

  • 新句子從底部滑入，舊句子往上推
  • 每個 cell 最多顯示 8 行，超過自動把最舊那行淡出
  • 每句話保留 60 秒後自動淡出消失
  • 越舊的句子越暗（最新最亮），方便視線聚焦

要改保留秒數：編輯 frontend/app.js 開頭的 LIFETIME_MS（毫秒）；要改每 cell 最多行數：改 MAX_LINES_PER_CELL。

切換語言

預設 zh，演唱會主持人若改用越南文，在 backend/.env 加一行：

WHISPER_LANGUAGE=vi

中越夾雜的場次才用 auto，但短句容易被誤判成韓文/日文。

原本 Hukou 專案的問題與本版改進

問題本版做法

疑難排解

找不到麥克風

跑一次 python -c "import sounddevice; print(sounddevice.query_devices())" 看看系統有哪些音訊輸入。如果想指定特定裝置，可以在 audio_capture.py 的 sd.RawInputStream(...) 加 device=N 參數。

Whisper 第一次啟動很慢

正常的。下載 small 模型約 250 MB，下載+載入大約 30 秒到 1 分鐘。下次啟動只剩載入時間，約 5~10 秒。模型會存在 backend/models/ 資料夾。

模型下載報錯 WinError 1314（沒有 symlink 權限）

這是 Windows 一般帳號不能建立符號連結造成的，與程式碼無關。本專案已改成把模型下載到 backend/models/ 並以「複製檔案」方式存放來避開。若仍遇到，三選一：用系統管理員身分執行 PowerShell；或開啟 Windows 開發人員模式（設定 → 隱私權與安全性 → 開發人員專用）；或更新 stt.py 到最新版。

webrtcvad 安裝失敗（要 C++ Build Tools）

Windows 上改裝預編譯版 webrtcvad-wheels，不要裝 webrtcvad。匯入名稱一樣，程式碼不用改。requirements.txt 已改用這個套件。

翻譯延遲還是太高

  1. 把 WHISPER_MODEL 從 small 改 base（速度快 2 倍但準確度略降）
  2. 把 WHISPER_DEVICE 改 cuda（如果有 NVIDIA 顯卡）
  3. 把 SILENCE_TIMEOUT_MS 從 700 降到 500（更快切句）
  4. 確認 Gemini 用的是 gemini-2.5-flash 而不是 pro 版

Gemini 回傳 429（rate limit）

免費版每分鐘有請求次數限制。演唱會場景建議升級到付費版（依量計費，便宜），或減少翻譯 worker 數量。

主持人說話很快會被切錯句

把 SILENCE_TIMEOUT_MS 提高到 1000~1200，避免太敏感切斷。
